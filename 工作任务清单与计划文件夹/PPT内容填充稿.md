## 幻灯片 4 — 内容页：为什么做这个项目

**标题：第三方依赖漏洞 —— 被忽视的安全短板**

左栏「痛点」：
- 现代 Java 应用 80% 以上代码来自第三方依赖，Log4Shell（CVE-2021-44228）级别漏洞随时可能爆发
- 人工排查依赖漏洞：逐个比对 CVE 库，耗时且极易遗漏
- 业界标准工具 OWASP Dependency-Check 是**命令行工具**：环境配置复杂、NVD 数据下载门槛高、结果不直观

右栏「方案定位」：
- 把 ODC 扫描引擎以 **library 方式**嵌入 Spring Boot，封装为 Web 平台
- **上传 ZIP → 一键扫描 → 可视化结果 → 三格式报告**，零命令行门槛
- 技术栈：Spring Boot 3.2.5 + MyBatis-Plus + H2 ｜ Vue 3 + Element Plus ｜ DC 引擎 9.0.0 ｜ 阿里云 OSS（条件装配）

> 💡 演讲词：选题源于集团真实痛点——依赖漏洞治理。我的定位不是重造扫描引擎，而是站在业界标准引擎之上，把"专家工具"变成"人人能用的平台"。

## 幻灯片 13（空白页 → 内容页）：技术难点攻坚

**标题：三个最有含金量的技术问题**

难点 1 — **NVD 数据源被 Cloudflare 拦截（最难）**
- 现象：API Key 有效、curl 直连 200，但 Java 客户端恒 520
- 定位：Cloudflare 按 **TLS 指纹**识别拦截 Java HTTP 客户端，换 IP/升级依赖均无效
- 破解：切换 DC 引擎 **datafeed 通道** + 镜像源 + 时间戳规范化预处理，最终实现**全离线扫描**

难点 2 — **@Async 异步任务失效排查**
- 同类调用不经过 Spring 代理导致异步失效 → 拆分 Executor 服务类 + 独立线程池隔离

难点 3 — **Zip Slip 路径穿越攻击防护**
- 恶意 ZIP 条目 `../../` 可逃逸上传目录 → 规范化路径 / 前缀校验 / 条目类型三层防护

> 💡 演讲词：重点讲难点 1 的排查过程（curl 能通、Java 不通 → 抓差异 → TLS 指纹 → 换协议通道），体现独立解决未知问题的能力。这也是面试官最可能追问的一页。
