analyzeDependencies()
  ├── initializeAndUpdateDatabase() → 更新 NVD 数据库
  ├── ensureDataExists() → 确保数据存在
  ├── 遍历所有 AnalysisPhase
  │   └── 遍历每个阶段的 Analyzer
  │       ├── initializeAnalyzer() → 初始化分析器
  │       ├── executeAnalysisTasks() → 多线程执行分析任务
  │       │   ├── getAnalysisTasks() → 为每个依赖创建 AnalysisTask
  │       │   ├── getExecutorService() → 获取线程池
  │       │   └── invokeAll() → 并行执行
  │       └── closeAnalyzer() → 关闭分析器
  └── 抛出 ExceptionCollection（如果有异常）
```

##### 分析阶段（AnalysisPhase）共15个阶段
| 阶段 | 代表分析器 | 作用 |
|------|-----------|------|
| INITIAL | ArchiveAnalyzer | 初始处理（解压归档文件） |
| PRE_INFORMATION_COLLECTION | ElixirMixAuditAnalyzer, RubyBundleAuditAnalyzer | 预信息收集 |
| INFORMATION_COLLECTION | JarAnalyzer, CentralAnalyzer, FileNameAnalyzer 等30+ | 收集依赖信息 |
| INFORMATION_COLLECTION2 | PEAnalyzer | 信息收集阶段2 |
| POST_INFORMATION_COLLECTION1~3 | DependencyMergingAnalyzer, HintAnalyzer, VersionFilterAnalyzer | 后处理 |
| PRE_IDENTIFIER_ANALYSIS | NpmCPEAnalyzer | 预标识符分析 |
| IDENTIFIER_ANALYSIS | CPEAnalyzer | CPE匹配 |
| POST_IDENTIFIER_ANALYSIS | CpeSuppressionAnalyzer, FalsePositiveAnalyzer | 后标识符分析 |
| FINDING_ANALYSIS | NvdCveAnalyzer, RetireJsAnalyzer, NodeAuditAnalyzer | 漏洞发现 |
| FINDING_ANALYSIS_PHASE2 | OssIndexAnalyzer | 漏洞发现阶段2 |
| POST_FINDING_ANALYSIS | KnownExploitedVulnerabilityAnalyzer, VulnerabilitySuppressionAnalyzer | 后漏洞分析 |
| FINAL | DependencyBundlingAnalyzer, UnusedSuppressionRuleAnalyzer | 最终处理 |

##### 关键设计模式
- **SPI（Service Provider Interface）**：通过 `ServiceLoader<Analyzer>` 动态加载所有分析器
- **策略模式**：每个分析器实现 `Analyzer` 接口，按阶段执行
- **多线程并行**：支持并行处理的分析器使用 `Executors.newFixedThreadPool(maxThreads)`
- **模板方法**：`AbstractFileTypeAnalyzer` 提供通用文件分析模板

##### analyzeDependency() 核心流程
```
analyzeDependency(dependency, engine)
  ├── collectClassNames() → 收集 JAR 中的类名
  ├── 检查是否为 macOS 元数据文件或无效 ZIP → 是则移除依赖
  ├── parseManifest() → 解析 MANIFEST.MF
  │   ├── 提取 Implementation-Title, Implementation-Version
  │   ├── 提取 Bundle-Name, Bundle-Version, Bundle-Description
  │   └── 过滤 IGNORE_KEYS 和 IGNORE_VALUES
  ├── analyzePOM() → 解析 pom.xml / pom.properties
  │   ├── 检查 JAR 同级目录是否有 .pom 文件
  │   ├── 遍历 JAR 内的 pom.xml 条目
  │   ├── 提取 GroupId, ArtifactId, Version
  │   ├── 设置 POM 证据（vendor, product, version）
  │   └── 发现 shaded 依赖 → 作为新依赖添加到 Engine

##### 关键设计

- **去重机制**：通过 SHA1 哈希判断是否已存在相同依赖
- **排除机制**：排除 macOS 元数据文件（`._` 前缀）和无效 ZIP 文件
