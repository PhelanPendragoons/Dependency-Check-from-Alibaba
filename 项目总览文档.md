## 一、项目背景

集团内部已累计数万款 Java、前端及多语言应用，第三方组件依赖数量呈指数级增长。近一年因第三方依赖漏洞导致的安全事件占比超过 **60%**，同时许可证不合规问题可能引发知识产权纠纷和重大商业损失。

目前集团内部缺乏统一的依赖分析工具和标准化使用文档，各团队使用的零散工具存在**功能不全、配置复杂、缺乏维护**等问题。

本项目基于业界成熟的开源安全引擎 OWASP Dependency-Check，通过深度分析、文档完善和轻量适配，打造一套可直接部署使用的内部解决方案。

## 六、技术亮点

### 6.1 OSS 条件装配 + 全链路降级

```java
@ConditionalOnProperty(prefix = "app.oss", name = "enabled", havingValue = "true")
```

- OSS 开启 → 报告自动上传至阿里云 OSS → 下载时从 OSS 获取
- OSS 关闭 → 报告存储在本地文件系统 → 零外部依赖
- 一套代码，配置即切换，无需改业务逻辑

### 6.2 Zip Slip 安全防护（三层设计）

```
第一层：路径穿越检测 → 解压时拒绝 ../ 路径
第二层：删除重试机制 → 100ms × 3 次，处理文件占用
第三层：事后验证     → Files.exists() 确认无残留
      + ZIP/项目目录分离 → 物理隔离防止连锁残留
```

### 6.3 异步任务线程池

- 专用线程池 `scanTaskExecutor`（core=2, max=4, queue=10）
- @Async 代理模式，服务层职责分离（Service + ExecutorService）
- 任务取消、进度跟踪（10%→90%→100%）、并发控制
- 前端自动轮询 RUNNING 状态（3 秒间隔，任务完成后自动停止）
