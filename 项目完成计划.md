## 一、项目全景概览

### 1.1 项目是做什么的？

集团内部有数万个Java、前端等应用，这些应用依赖了大量的第三方组件（比如Spring框架、Log4j日志库等）。这些第三方组件可能存在**安全漏洞**（比如Log4j的"核弹级"漏洞）或**许可证问题**（比如用了GPL协议的库可能导致商业纠纷）。

本项目就是要做一个**内部工具平台**，让开发人员上传自己的项目代码，系统自动扫描出所有依赖的安全漏洞和许可证问题，并生成报告。

#### 1.2 配置Maven阿里云镜像

##### 为什么要配镜像？

Maven默认从 **Maven Central（中央仓库）** 下载项目依赖的JAR包，但这个仓库的服务器在国外。国内下载速度非常慢（可能只有几十KB/s），而且经常超时失败。

阿里云在国内提供了Maven镜像服务，速度可以达到几MB/s。**配置镜像后，Maven会从阿里云下载依赖，而不是从中央仓库。**

##### 第一步：找到settings.xml文件

Maven的配置文件叫 `settings.xml`，有两个位置：

| 位置 | 路径（Windows） | 说明 |
|------|----------------|------|
| **用户级别（推荐）** | `C:\Users\你的用户名\.m2\settings.xml` | 只对当前用户生效，推荐在这里配置 |
| 全局级别 | `D:\apache-maven-3.8.x\conf\settings.xml` | 对所有用户生效，需要管理员权限 |

> **注意**：`.m2` 文件夹默认是隐藏的，需要在文件资源管理器中开启"查看隐藏的项目"。

##### 第二步：检查文件是否存在

打开文件资源管理器，导航到 `C:\Users\你的用户名\.m2\` 目录：

- 如果已经有 `settings.xml` 文件 → 直接用记事本/VS Code打开编辑
- 如果没有 `settings.xml` 文件 → 需要手动创建一个

##### 第三步：创建/编辑settings.xml

用VS Code或记事本打开 `settings.xml`，将以下内容**完整复制**进去：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0
                              http://maven.apache.org/xsd/settings-1.0.0.xsd">

    <!-- 配置阿里云镜像 -->
    <mirrors>
        <mirror>
            <id>aliyunmaven</id>
            <name>阿里云公共仓库</name>
            <!-- mirrorOf * 表示所有依赖都从阿里云下载 -->
            <mirrorOf>*</mirrorOf>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
    </mirrors>

    <!-- （可选）配置JDK版本，避免编译时提示"不再支持源选项 5" -->
    <profiles>
        <profile>
            <id>jdk-17</id>
            <activation>
                <activeByDefault>true</activeByDefault>
                <jdk>17</jdk>
            </activation>
            <properties>
                <maven.compiler.source>17</maven.compiler.source>
                <maven.compiler.target>17</maven.compiler.target>
                <maven.compiler.compilerVersion>17</maven.compiler.compilerVersion>
            </properties>
        </profile>
    </profiles>

</settings>
```

##### 第四步：验证配置是否生效

打开命令行（CMD或PowerShell），执行：

```bash
# 查看Maven版本和settings.xml位置
mvn -version
```

如果配置成功，输出中会显示：
```
...
User settings file: C:\Users\你的用户名\.m2\settings.xml
...
```

然后尝试编译一个项目，观察下载速度：

```bash
# 进入项目目录后执行
mvn clean install -DskipTests
```

如果看到下载速度达到 **1MB/s以上**，说明配置成功！

##### 第五步：如果下载仍然很慢怎么办？

如果配置后下载速度仍然不理想，可以尝试以下方案：

**方案一：使用阿里云其他镜像仓库**

```xml
<mirrors>
    <!-- 阿里云中央仓库镜像 -->
    <mirror>
        <id>aliyun-central</id>
        <mirrorOf>central</mirrorOf>
        <name>阿里云中央仓库镜像</name>
        <url>https://maven.aliyun.com/repository/central</url>
    </mirror>
    <!-- 阿里云Spring插件镜像 -->
    <mirror>
        <id>aliyun-spring</id>
        <mirrorOf>spring</mirrorOf>
        <name>阿里云Spring仓库镜像</name>
        <url>https://maven.aliyun.com/repository/spring</url>
    </mirror>
</mirrors>
```

**方案二：在项目pom.xml中直接配置仓库（不依赖settings.xml）**

```xml
<repositories>
    <repository>
        <id>aliyun</id>
        <name>阿里云仓库</name>
        <url>https://maven.aliyun.com/repository/public</url>
    </repository>
</repositories>
```

**方案三：使用华为云镜像（阿里云镜像的备选）**

```xml
<mirror>
    <id>huaweicloud</id>
    <mirrorOf>*</mirrorOf>
    <name>华为云镜像</name>
    <url>https://repo.huaweicloud.com/repository/maven/</url>
</mirror>
```

##### 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| `Could not transfer artifact` | 网络连接失败 | 检查网络，或切换为华为云镜像 |
| `settings.xml` 报错 | XML格式错误 | 检查标签是否闭合，不要有中文标点 |
| 配置后仍然从中央仓库下载 | settings.xml位置不对 | 确认文件在 `C:\Users\用户名\.m2\` 目录下 |
| 下载速度还是很慢 | 可能是公司网络限制 | 尝试使用VPN，或联系IT部门开放Maven仓库访问 |

###### ❌ 情况一：提示 `'node' 不是内部或外部命令`

**原因**：Node.js没有安装，或者安装时没有勾选"Add to PATH"（添加到环境变量）。

**解决方法（按顺序尝试）：**

**方法A：检查Node.js是否已安装（推荐新手）**

1. 打开文件资源管理器，导航到 `C:\Program Files\nodejs\`
2. 看看这个文件夹里有没有 `node.exe` 文件
3. **如果有 `node.exe`** → 说明已安装，只是没添加到PATH，跳到下面的"手动添加环境变量"
4. **如果没有 `node.exe`** → 说明没安装，跳到"重新安装Node.js"

**方法B：手动添加环境变量（如果已安装但没加到PATH）**

1. 右键"此电脑" → **属性** → **高级系统设置** → **环境变量**
2. 在"系统变量"中找到 **Path**，双击打开
3. 点击 **新建**，添加Node.js的安装路径（通常是 `C:\Program Files\nodejs\`）
4. 点击 **确定** 保存
5. **关闭当前CMD窗口，重新打开**，再次输入 `node -v`

**方法C：重新安装Node.js（最稳妥）**

1. 访问 https://nodejs.org/ 下载 **Node.js 18.x LTS** 版本
2. 运行安装程序，一路点击 Next
3. **关键步骤**：在安装向导中，看到如下界面时：

```
Custom Setup
Select the way you want features to be installed.

☑ Node.js runtime
☑ npm package manager
☑ Add to PATH          ← ← ← 这一项必须勾选上！
☐ Online documentation shortcuts
```

4. 确保 **"Add to PATH"** 前面是 ☑ 勾选状态
5. 点击 Next → Install，等待安装完成
6. **关闭当前CMD窗口，重新打开**，再次输入 `node -v`

###### ❌ 情况二：`node -v` 能显示版本，但 `npm -v` 失败

**原因**：Node.js安装成功，但npm没有正确安装。

**解决方法：**

```bash
# 方法1：手动安装npm（在命令行中执行）

# 方法2：如果方法1不行，用Node.js自带的工具修复

# 然后在该目录下打开命令行，执行：
cd "C:\Program Files\nodejs"
npm install -g npm

# 方法3：如果还不行，卸载Node.js重新安装
```

5. 点击 **确定** 保存
6. **关闭当前CMD窗口，重新打开**，再次输入 `npm -v`

###### ❌ 情况四：PowerShell提示 `无法加载文件 npm.ps1，因为在此系统上禁止运行脚本`

**错误信息：**
```
npm : 无法加载文件 C:\Program Files\nodejs\npm.ps1，因为在此系统上禁止运行脚本。
```

**原因**：这是Windows PowerShell的安全策略（Execution Policy）问题，不是Node.js没安装好。PowerShell默认禁止运行 `.ps1` 脚本文件。

**解决方法一（推荐，最简单）：改用CMD（命令提示符）**

Node.js其实已经安装成功了，只是PowerShell的限制。换成CMD就好了：

1. 按 `Win + R` 键
2. 输入 `cmd`，回车
3. 在CMD窗口中输入：
```bash
node -v
```

## 阶段二：基准引擎深度分析（第3-4周）

### 第3周：理解DependencyCheck核心架构

#### 3.1 核心引擎入口：Engine.java

`Engine.java` 是整个DependencyCheck的**大脑**，位于 `core/src/main/java/org/owasp/dependencycheck/Engine.java`。

它的核心工作流程：

```
1. 初始化阶段
   ├── 加载配置（Settings）
   ├── 发现所有分析器（AnalyzerService）
   └── 打开漏洞数据库连接（CveDB）

2. 扫描阶段（scan()）
   ├── 递归扫描指定目录
   ├── 根据文件类型匹配分析器
   └── 构建依赖树（Dependency）

3. 分析阶段（analyze()）
   ├── 按阶段执行所有分析器
   │   ├── INITIAL → 初始处理
   │   ├── INFORMATION_COLLECTION → 收集依赖信息
   │   ├── IDENTIFIER_ANALYSIS → 识别依赖标识
   │   ├── FINDING_ANALYSIS → 匹配漏洞
   │   └── FINAL → 最终处理
   └── 每个分析器处理特定类型的文件

4. 报告生成阶段（generateReport()）
   └── ReportGenerator生成HTML/XML/JSON报告
```

> **给实习生的理解**：可以把Engine想象成一个工厂流水线：
> 1. 先把项目文件"拆开"（扫描）
> 2. 然后逐个"检查"每个零件（分析）
> 3. 最后"打印"检查报告（报告生成）

#### 3.2 分析器体系（60+分析器）

分析器是DependencyCheck最核心的部分，每个分析器负责解析一种类型的文件。

**关键分析器分类：**

| 分析器类别 | 代表分析器 | 作用 |
|-----------|-----------|------|
| Java相关 | `JarAnalyzer` | 解析JAR包中的pom.xml、MANIFEST.MF |
| Maven相关 | `PinnedMavenInstallAnalyzer` | 解析Maven安装的依赖 |
| NPM相关 | `NodePackageAnalyzer` | 解析package.json |
| Python相关 | `PipAnalyzer`、`PipfileAnalyzer` | 解析Python依赖 |
| 漏洞匹配 | `CPEAnalyzer` | 将依赖与CPE（通用平台枚举）匹配 |
| 漏洞匹配 | `NvdCveAnalyzer` | 从NVD数据库匹配CVE漏洞 |
| 许可证分析 | 内置于多个分析器中 | 自动识别许可证类型 |
| 报告生成 | `FalsePositiveAnalyzer` | 过滤误报 |

**分析器执行顺序（AnalysisPhase）：**

```java
public enum AnalysisPhase {
    INITIAL,                    // 初始阶段
    INFORMATION_COLLECTION,     // 信息收集阶段1
    INFORMATION_COLLECTION2,    // 信息收集阶段2
    IDENTIFIER_ANALYSIS,        // 标识符分析
    POST_IDENTIFIER_ANALYSIS,   // 标识符后分析
    FINDING_ANALYSIS,           // 漏洞发现分析
    FINDING_ANALYSIS_PHASE2,    // 漏洞发现分析阶段2
    POST_FINDING_ANALYSIS,      // 漏洞发现后分析
    FINAL                       // 最终阶段
}
```


#### 3.3 漏洞数据源

| 数据源 | 全称 | 说明 |
|-------|------|------|
| NVD | National Vulnerability Database | 美国国家漏洞数据库，最权威的漏洞数据源 |
| OSV | Open Source Vulnerabilities | 开源漏洞数据库，覆盖更广 |
| RetireJS | Retire.js | 专门针对JavaScript库的漏洞库 |
| Known Exploited Vulnerabilities | CISA KEV | 美国CISA发布的已知被利用漏洞目录 |

#### 3.4 报告生成

`ReportGenerator.java` 支持生成多种格式的报告。**默认格式为 HTML**，如果不指定 `--format` 参数，会自动生成 HTML 报告。

| 格式 | 命令参数 | 类/方法 | 说明 |
|------|---------|---------|------|
| **HTML** ✅ **默认** | `--format HTML` 或不指定 | `generateReport()` | 最常用的可视化报告，浏览器可直接打开 |
| XML | `--format XML` | `generateXmlReport()` | 机器可读的XML格式 |
| JSON | `--format JSON` | `generateJsonReport()` | 机器可读的JSON格式 |
| SARIF | `--format SARIF` | `generateSarifReport()` | 静态分析结果交换格式（可用于CI/CD） |
| CSV | `--format CSV` | `generateCsvReport()` | 表格数据格式，可用Excel打开 |
| JUNIT | `--format JUNIT` | `generateJunitXmlReport()` | 单元测试报告格式（可用于Jenkins等） |
| ALL | `--format ALL` | 同时生成所有格式 | 一次性输出全部格式的报告 |

> **注意**：DependencyCheck **不支持原生 PDF 输出**。如果项目需要 PDF 格式的报告，需要通过 HTML 转 PDF 的方式实现（例如使用 wkhtmltopdf 或 Puppeteer）。

**同时生成多种格式：**
```bash
dependency-check --scan /path/to/project \
  --format HTML \
  --format JSON \
  --format CSV \
  --out ./reports
```

#### B1. 封装DependencyCheck引擎（ScanEngineService）

这是**最核心的类**，负责封装DependencyCheck的Engine API。

```java
package com.alibaba.dependencycheck.service;

import lombok.extern.slf4j.Slf4j;
import org.owasp.dependencycheck.Engine;
import org.owasp.dependencycheck.data.nvdcve.CveDB;
import org.owasp.dependencycheck.reporting.ReportGenerator;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;

import java.io.File;
import java.util.List;

/**
 * 封装DependencyCheck引擎的核心服务类
 */
@Slf4j
@Service
public class ScanEngineService {

    @Value("${dependency-check.data-directory}")
    private String dataDirectory;

    /**
     * 执行扫描
     *
     * @param scanPath 待扫描的文件/目录路径
     * @param reportDir 报告输出目录
     * @return 扫描到的依赖列表
     */
    public List<org.owasp.dependencycheck.dependency.Dependency> scan(String scanPath, String reportDir) {
        Engine engine = null;
        try {
            // 1. 创建Engine实例
            //    注意：Engine的构造函数会自动加载配置、发现分析器、打开数据库连接
            engine = new Engine(Engine.mode(true));  // true = 使用本地数据库模式

            // 2. 执行扫描
            //    scan()方法会递归扫描指定目录，根据文件类型匹配分析器
            log.info("开始扫描: {}", scanPath);
            engine.scan(scanPath);

            // 3. 执行分析
            //    analyze()方法会按阶段执行所有分析器
            //    包括：信息收集 → 标识符分析 → 漏洞匹配 → 最终处理
            engine.analyzeDependencies();

            // 4. 获取扫描结果
            List<org.owasp.dependencycheck.dependency.Dependency> dependencies = engine.getDependencies();
            log.info("扫描完成，发现 {} 个依赖", dependencies.size());

            // 5. 生成HTML报告
            //    使用writeReports()方法生成报告（v12.0.2的API）
            //    注意：旧版本（v9.0.0）使用的是generateReport()，新版本已废弃
            File reportDirectory = new File(reportDir);
            if (!reportDirectory.exists()) {
                reportDirectory.mkdirs();
            }
            engine.writeReports(scanPath, reportDirectory, ReportGenerator.Format.HTML);

            return dependencies;

        } catch (Exception e) {
            log.error("扫描失败", e);
            throw new RuntimeException("扫描失败: " + e.getMessage(), e);
        } finally {
            // 6. 释放资源
            //    必须调用close()方法，否则数据库连接不会释放
            if (engine != null) {
                engine.close();
            }
        }
    }
}
```

> ⚠️ **API变更说明**：
> - **DependencyCheck v9.0.0**：使用 `engine.generateReport(path, directory, format)` 生成报告
> - **DependencyCheck v12.0.2**：使用 `engine.writeReports(path, directory, format)` 生成报告（`generateReport` 已废弃）
> - 本项目的pom.xml配置的是v12.0.2，所以代码中使用 `writeReports()`

##### AsyncConfig.java - 异步任务线程池配置

```java
package com.alibaba.dependencycheck.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import java.util.concurrent.Executor;

/**
 * 异步任务配置
 * 用于执行耗时的扫描任务，避免阻塞HTTP请求线程
 */
@Configuration
@EnableAsync
public class AsyncConfig {

    @Bean("scanTaskExecutor")
    public Executor scanTaskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(2);        // 核心线程数
        executor.setMaxPoolSize(4);         // 最大线程数
        executor.setQueueCapacity(10);      // 队列容量
        executor.setThreadNamePrefix("scan-task-");  // 线程名前缀
        executor.setWaitForTasksToCompleteOnShutdown(true);
        executor.setAwaitTerminationSeconds(60);
        executor.initialize();
        return executor;
    }
}
```

##### ScanTaskService.java - 扫描任务管理

```java
package com.alibaba.dependencycheck.service;

import com.alibaba.dependencycheck.model.entity.ScanTask;
import com.alibaba.dependencycheck.model.entity.ScanResult;
import com.alibaba.dependencycheck.repository.ScanTaskRepository;
import com.alibaba.dependencycheck.repository.ScanResultRepository;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.scheduling.annotation.Async;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;
import java.util.List;

/**
 * 扫描任务管理服务
 * 负责创建、执行、跟踪扫描任务
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class ScanTaskService {

    private final ScanEngineService scanEngineService;
    private final ScanTaskRepository scanTaskRepository;
    private final ScanResultRepository scanResultRepository;

    /**
     * 异步执行扫描任务
     * 使用@Async注解，方法会在独立的线程池中执行
     */
    @Async("scanTaskExecutor")
    public void executeScan(Long taskId, String scanPath, String reportDir) {
        ScanTask task = scanTaskRepository.findById(taskId)
                .orElseThrow(() -> new RuntimeException("任务不存在: " + taskId));

        try {
            // 更新任务状态为"运行中"
            task.setStatus("RUNNING");
            task.setStartedAt(LocalDateTime.now());
            scanTaskRepository.save(task);

            // 执行扫描
            List<org.owasp.dependencycheck.dependency.Dependency> dependencies =
                    scanEngineService.scan(scanPath, reportDir);

            // 保存扫描结果
            for (org.owasp.dependencycheck.dependency.Dependency dep : dependencies) {
                ScanResult result = convertToScanResult(taskId, dep);
                scanResultRepository.save(result);
            }

            // 更新任务状态为"完成"
            task.setStatus("COMPLETED");
            task.setProgress(100);
            task.setTotalDependencies(dependencies.size());
            task.setCompletedAt(LocalDateTime.now());
            scanTaskRepository.save(task);

            log.info("扫描任务 {} 完成", taskId);

        } catch (Exception e) {
            log.error("扫描任务 {} 失败", taskId, e);
            task.setStatus("FAILED");
            task.setErrorMessage(e.getMessage());
            task.setCompletedAt(LocalDateTime.now());
            scanTaskRepository.save(task);
        }
    }

    /**
     * 将DependencyCheck的Dependency对象转换为自定义的ScanResult实体
     */
    private ScanResult convertToScanResult(Long taskId,
            org.owasp.dependencycheck.dependency.Dependency dep) {

        ScanResult result = new ScanResult();
        result.setTaskId(taskId);
        result.setDependencyName(dep.getDisplayName());
        result.setDependencyVersion(dep.getVersion());
        result.setFilePath(dep.getFilePath());

        // 获取漏洞信息
        // 注意：v12.0.2中使用 getVulnerabilities() 而不是 getVulnerableSoftware()
        for (org.owasp.dependencycheck.dependency.Vulnerability vuln : dep.getVulnerabilities()) {
            result.setCveId(vuln.getName());
            result.setCvssScore(vuln.getCvssV2().getScore());
            result.setSeverity(vuln.getSeverity());
            result.setDescription(vuln.getDescription());
            result.setIsVulnerable(true);
        }

        // 获取许可证信息
        for (org.owasp.dependencycheck.dependency.License license : dep.getLicenses()) {
            result.setLicenseName(license.getName());
            result.setLicenseUrl(license.getUrl());
        }

        return result;
    }
}
```

```java
package com.alibaba.dependencycheck.service;

import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;
import org.springframework.web.multipart.MultipartFile;

import java.io.File;
import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.util.zip.ZipEntry;
import java.util.zip.ZipInputStream;

/**
 * 项目管理服务
 * 负责处理项目文件的上传、解压和存储
 */
@Slf4j
@Service
public class ProjectService {

    @Value("${app.upload-dir:./uploads}")
    private String uploadDir;

    /**
     * 上传并解压项目文件
     *
     * @param file 上传的ZIP文件
     * @param projectName 项目名称
     * @return 解压后的目录路径
     */
    public String uploadAndExtract(MultipartFile file, String projectName) throws IOException {
        // 1. 创建项目目录
        String projectDir = uploadDir + File.separator + projectName;
        Path projectPath = Paths.get(projectDir);
        if (!Files.exists(projectPath)) {
            Files.createDirectories(projectPath);
        }

        // 2. 保存上传的ZIP文件
        String zipPath = projectDir + File.separator + file.getOriginalFilename();
        file.transferTo(new File(zipPath));

        // 3. 解压ZIP文件
        unzip(zipPath, projectDir);

        // 4. 删除ZIP文件（可选）
        // new File(zipPath).delete();

        return projectDir;
    }

    /**
     * 解压ZIP文件
     */
    private void unzip(String zipFilePath, String destDir) throws IOException {
        File dest = new File(destDir);
        if (!dest.exists()) {
            dest.mkdirs();
        }

        try (ZipInputStream zis = new ZipInputStream(
                Files.newInputStream(Paths.get(zipFilePath)))) {
            ZipEntry entry;
            while ((entry = zis.getNextEntry()) != null) {
                String filePath = destDir + File.separator + entry.getName();
                if (entry.isDirectory()) {
                    new File(filePath).mkdirs();
                } else {
                    // 确保父目录存在
                    new File(filePath).getParentFile().mkdirs();
                    // 写入文件
                    Files.copy(zis, Paths.get(filePath));
                }
                zis.closeEntry();
            }
        }
    }
}
```

```java
package com.alibaba.dependencycheck.model.vo;

import lombok.Data;

/**
 * 统一API返回结果
 */
@Data
public class Result<T> {
    private int code;       // 状态码（200=成功）
    private String message; // 提示信息
    private T data;         // 返回数据

    private Result(int code, String message, T data) {
        this.code = code;
        this.message = message;
        this.data = data;
    }

    public static <T> Result<T> success() {
        return new Result<>(200, "操作成功", null);
    }

    public static <T> Result<T> success(T data) {
        return new Result<>(200, "操作成功", data);
    }

    public static <T> Result<T> error(int code, String message) {
```

#### D2. Excel报告生成（Apache POI）

```java
package com.alibaba.dependencycheck.service;

import com.alibaba.dependencycheck.model.entity.ScanResult;
import com.alibaba.dependencycheck.repository.ScanResultRepository;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.apache.poi.ss.usermodel.*;
import org.apache.poi.xssf.usermodel.XSSFWorkbook;
import org.springframework.stereotype.Service;

import java.io.ByteArrayOutputStream;
import java.util.List;

/**
 * Excel报告生成服务
 * 使用Apache POI生成.xlsx格式的扫描报告
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class ExcelReportService {

    private final ScanResultRepository scanResultRepository;

    /**
     * 生成Excel报告
     *
     * @param taskId 扫描任务ID
     * @return Excel文件的字节数组
     */
    public byte[] generateExcelReport(Long taskId) {
        List<ScanResult> results = scanResultRepository.findByTaskId(taskId);

        try (Workbook workbook = new XSSFWorkbook()) {
            Sheet sheet = workbook.createSheet("依赖扫描报告");

            // 创建标题行
            String[] headers = {"依赖名称", "版本", "CVE编号", "CVSS评分",
                    "严重等级", "漏洞描述", "许可证名称"};
            Row headerRow = sheet.createRow(0);
            for (int i = 0; i < headers.length; i++) {
                Cell cell = headerRow.createCell(i);
                cell.setCellValue(headers[i]);
                CellStyle style = workbook.createCellStyle();
                Font font = workbook.createFont();
                font.setBold(true);
                style.setFont(font);
                cell.setCellStyle(style);
            }

            // 填充数据行
            int rowNum = 1;
            for (ScanResult result : results) {
                Row row = sheet.createRow(rowNum++);
                row.createCell(0).setCellValue(result.getDependencyName());
                row.createCell(1).setCellValue(result.getDependencyVersion());
                row.createCell(2).setCellValue(result.getCveId());
                row.createCell(3).setCellValue(
                        result.getCvssScore() != null ? result.getCvssScore().doubleValue() : 0);
                row.createCell(4).setCellValue(result.getSeverity());
                row.createCell(5).setCellValue(result.getDescription());
                row.createCell(6).setCellValue(result.getLicenseName());
            }

            // 自动调整列宽
            for (int i = 0; i < headers.length; i++) {
                sheet.autoSizeColumn(i);
            }

            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            workbook.write(bos);
            return bos.toByteArray();

        } catch (Exception e) {
            log.error("生成Excel报告失败", e);
            throw new RuntimeException("生成Excel报告失败", e);
        }
    }
}
```

**PDF生成服务：**

```java
package com.alibaba.dependencycheck.service;

import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

import java.io.BufferedReader;
import java.io.InputStreamReader;

/**
 * PDF报告生成服务
 * 使用wkhtmltopdf将HTML报告转换为PDF
 */
@Slf4j
@Service
public class PdfReportService {

    /**
     * 将HTML文件转换为PDF
     *
     * @param htmlPath HTML文件路径
     * @param pdfPath  PDF输出路径
     */
    public void convertHtmlToPdf(String htmlPath, String pdfPath) {
        try {
            // 构建wkhtmltopdf命令
            ProcessBuilder pb = new ProcessBuilder(
                    "wkhtmltopdf",
                    "--enable-local-file-access",  // 允许访问本地文件
                    "--page-size", "A4",
                    "--margin-top", "10mm",
                    "--margin-bottom", "10mm",
                    htmlPath,
                    pdfPath
            );

            Process process = pb.start();

            // 读取错误输出
            try (BufferedReader reader = new BufferedReader(
                    new InputStreamReader(process.getErrorStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {
                    log.debug("wkhtmltopdf: {}", line);
                }
            }

            int exitCode = process.waitFor();
            if (exitCode != 0) {
                throw new RuntimeException("wkhtmltopdf执行失败，退出码: " + exitCode);
            }

            log.info("PDF报告生成成功: {}", pdfPath);

        } catch (Exception e) {
            log.error("PDF报告生成失败", e);
            throw new RuntimeException("PDF报告生成失败", e);
        }
    }
}
```

> - **Apache PDFBox**：Apache开源的PDF库，功能相对基础

##### ReportService.java - 报告存储与下载服务

```java
package com.alibaba.dependencycheck.service;

import com.alibaba.dependencycheck.model.entity.ScanTask;
import com.alibaba.dependencycheck.repository.ScanTaskRepository;
import com.aliyun.oss.OSS;
import com.aliyun.oss.model.GetObjectRequest;
import com.aliyun.oss.model.PutObjectRequest;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.core.io.ByteArrayResource;
import org.springframework.core.io.Resource;
import org.springframework.stereotype.Service;

import java.io.ByteArrayOutputStream;
import java.io.File;

/**
 * 报告管理服务
 * 负责报告的生成、上传OSS、下载
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class ReportService {

    private final OSS ossClient;
    private final ScanTaskRepository scanTaskRepository;
    private final ExcelReportService excelReportService;
    private final PdfReportService pdfReportService;

    @Value("${aliyun.oss.bucket-name}")
    private String bucketName;

    @Value("${aliyun.oss.report-path}")
    private String reportPath;

    /**
     * 上传报告到OSS
     *
     * @param taskId 扫描任务ID
     * @param localFilePath 本地报告文件路径
     * @param ossFileName OSS中的文件名
     */
    public void uploadReportToOss(Long taskId, String localFilePath, String ossFileName) {
        try {
            String ossKey = reportPath + taskId + "/" + ossFileName;
            PutObjectRequest request = new PutObjectRequest(bucketName, ossKey, new File(localFilePath));
            ossClient.putObject(request);
            log.info("报告已上传到OSS: {}", ossKey);
        } catch (Exception e) {
            log.error("上传报告到OSS失败", e);
            throw new RuntimeException("上传报告到OSS失败", e);
        }
    }

    /**
     * 从OSS下载HTML报告
     */
    public Resource getHtmlReport(Long taskId) {
        return downloadFromOss(taskId, "dependency-check-report.html");
    }

    /**
     * 生成并获取Excel报告
     */
    public Resource getExcelReport(Long taskId) {
        byte[] excelBytes = excelReportService.generateExcelReport(taskId);
        return new ByteArrayResource(excelBytes);
    }

    /**
     * 生成并获取PDF报告
     */
    public Resource getPdfReport(Long taskId) {
        // 1. 先获取本地HTML报告路径
        ScanTask task = scanTaskRepository.findById(taskId)
                .orElseThrow(() -> new RuntimeException("任务不存在"));
        String htmlPath = "./reports/" + taskId + "/dependency-check-report.html";
        String pdfPath = "./reports/" + taskId + "/dependency-check-report.pdf";

        // 2. 将HTML转换为PDF
        pdfReportService.convertHtmlToPdf(htmlPath, pdfPath);

        // 3. 返回PDF文件
        return downloadFromOss(taskId, "dependency-check-report.pdf");
    }

    /**
     * 从OSS下载文件
     */
    private Resource downloadFromOss(Long taskId, String fileName) {
        try {
            String ossKey = reportPath + taskId + "/" + fileName;
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ossClient.getObject(new GetObjectRequest(bucketName, ossKey), bos);
            return new ByteArrayResource(bos.toByteArray());
        } catch (Exception e) {
            log.error("从OSS下载报告失败", e);
            throw new RuntimeException("下载报告失败", e);
        }
    }
}
```

> ⚠️ **注意**：
> - 阿里云OSS的 `OSSClient` 在 v3.17.4 版本中仍然可用，但阿里云推荐使用 `OSSAsyncClient` 或新的 `OSS` 接口
> - 生产环境中建议使用 `@PreDestroy` 注解在应用关闭时调用 `ossClient.shutdown()` 释放资源
> - OSS的AccessKey和SecretKey应该通过环境变量或阿里云RAM角色授权，**不要硬编码在代码中**
> - 本项目的application.yml中已经配置了通过环境变量注入的方式：`${OSS_ACCESS_KEY_ID:}`

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| NVD API Key申请困难 | 漏洞数据库下载慢 | 使用NVD镜像或提前下载数据库文件 |
| DependencyCheck编译失败 | 无法获取核心引擎 | 使用Maven中央仓库的正式发布版本 |
| 扫描时间过长 | 用户体验差 | 使用异步任务+进度提示 |
| 大文件上传失败 | 无法扫描大型项目 | 配置大文件上传限制，支持分片上传 |
| OSS配置复杂 | 报告存储不可用 | 先使用本地文件存储，后期再集成OSS |
| 前端开发经验不足 | 页面开发进度慢 | 使用Element Plus组件库快速搭建 |
