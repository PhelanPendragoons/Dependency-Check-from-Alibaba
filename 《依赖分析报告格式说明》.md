## 1. 概述

本平台为每次扫描任务生成三种格式的分析报告，覆盖不同的使用场景：

| 格式 | 生成技术 | 扩展名 | 查看方式 |
|------|----------|--------|----------|
| **HTML** | OWASP Dependency-Check 引擎原生 | `.html` | 浏览器直接打开 |
| **Excel** | Apache POI 5.2.5 | `.xlsx` | Microsoft Excel / WPS |
| **PDF** | Flying Saucer 9.4.0（HTML 转 PDF） | `.pdf` | PDF 阅读器 / 浏览器 |

### 统一报告内容

三种格式报告均包含以下核心信息：

1. **扫描摘要**：项目名称、扫描时间、依赖总数、漏洞依赖数
2. **漏洞清单**：CVE 编号、CVSS 评分、严重等级（CRITICAL/HIGH/MEDIUM/LOW）、影响组件及版本、漏洞描述
3. **依赖清单**：所有已解析的依赖包名称、版本、文件路径
4. **许可证信息**：依赖组件的开源许可证类型

### 报告文件结构

```
./reports/
└── {taskId}/
    ├── dependency-check-report.html       ← HTML 报告
    ├── dependency-check-report-{id}.xlsx  ← Excel 报告
    └── dependency-check-report-{id}.pdf   ← PDF 报告
```

## 2. HTML 报告

### 2.1 生成方式

HTML 报告由 **OWASP Dependency-Check 引擎直接生成**，调用 `Engine.writeReports()` 方法输出。这是引擎的原生输出格式，信息最完整、最详细。

### 2.3 特性

| 特性 | 说明 |
|------|------|
| 交互式导航 | 可点击目录跳转到对应章节 |

- 安全工程师分析漏洞详情

### 2.5 API 访问

```
GET /api/reports/{taskId}?format=html
Content-Type: text/html
Content-Disposition: inline
```

## 3. Excel 报告

### 3.1 生成方式

Excel 报告由后端使用 **Apache POI** 库生成 `.xlsx` 格式，程序化创建三个工作表，每个工作表带有专业商务风格的样式。

详细实现参见 `ReportService.generateExcelReport()`。

### 3.2 三个工作表

#### 工作表 1：概览（Summary）

```
┌─────────────────────────────────────┐
│       依赖安全扫描报告               │  ← 深蓝背景 + 白色加粗 16pt
├────────────────┬────────────────────┤
│ 项目名称        │ 5                  │
│ 扫描时间        │ 2026-07-14 17:30  │
│ 总依赖数        │ 234               │
│ 漏洞依赖数      │ 8                 │
│ 扫描状态        │ COMPLETED         │
└────────────────┴────────────────────┘
```

样式特点：
- 标题行：深蓝色背景、白色加粗 16pt 字体、水平居中
- 数据区域：细边框、自动换行、垂直居中

#### 工作表 2：漏洞详情（Vulnerabilities）

```
┌──────────────┬──────────┬──────────────┬──────────┬──────────┬──────────┐
│ 依赖名称      │ 版本     │ CVE 编号      │ CVSS 评分 │ 严重等级  │ 漏洞描述  │  ← 蓝底白字加粗
├──────────────┼──────────┼──────────────┼──────────┼──────────┼──────────┤
│ spring-beans │ 5.3.20   │ CVE-2022-22965│ 9.8     │ 🔴CRITICAL│ RCE...   │  ← 红底
│ log4j-core   │ 2.14.1   │ CVE-2021-44228│ 10.0    │ 🔴HIGH    │ RCE...   │  ← 红底
│ jackson      │ 2.13.2   │ CVE-2022-42003│ 7.5     │ 🟠MEDIUM  │ DoS...   │  ← 橙底
│ guava        │ 30.1     │ CVE-2023-2976 │ 3.7     │ 🟢LOW     │ Info...  │  ← 黄底
└──────────────┴──────────┴──────────────┴──────────┴──────────┴──────────┘
```

列说明：

| 列 | 内容来源 | 数据库字段 |
|----|----------|------------|
| 依赖名称 | 解析到的依赖包名 | `scan_result.dependency_name` |
| 版本 | 依赖版本号 | `scan_result.dependency_version` |
| CVE 编号 | 匹配到的 CVE | `scan_result.cve_id` |
| CVSS 评分 | 0.0–10.0 | `scan_result.cvss_score` |
| 严重等级 | CRITICAL / HIGH / MEDIUM / LOW | `scan_result.severity` |
| 漏洞描述 | CVE 官方描述 | `scan_result.description` |

**严重等级条件格式**：

| 等级 | 单元格背景色 | 字体 |
|------|-------------|------|
| CRITICAL | 红色 + 加粗 | 白色（突出显示最危险项） |
| HIGH | 红色 + 加粗 | 白色 |
| MEDIUM | 橙色 + 加粗 | 黑色 |
| LOW | 黄色 + 加粗 | 黑色 |

> 💡 仅包含 `is_vulnerable = true` 的条目，非漏洞依赖不出现在此工作表。

#### 工作表 3：依赖清单（Dependencies）

```
┌──────────────┬──────────┬────────────────────────┬────────────┬──────────┐
│ 依赖名称      │ 版本     │ 文件路径               │ 许可证       │ 是否有漏洞 │
├──────────────┼──────────┼────────────────────────┼────────────┼──────────┤
│ spring-beans │ 5.3.20   │ /BOOT-INF/lib/...     │ Apache 2.0 │ 是       │
│ log4j-core   │ 2.14.1   │ /BOOT-INF/lib/...     │ Apache 2.0 │ 是       │
│ jackson      │ 2.13.2   │ /BOOT-INF/lib/...     │ Apache 2.0 │ 是       │
│ guava        │ 30.1     │ /BOOT-INF/lib/...     │ Apache 2.0 │ 否       │
│ ...          │ ...      │ ...                   │ ...        │ ...      │
└──────────────┴──────────┴────────────────────────┴────────────┴──────────┘
```

> 💡 依赖名称+版本去重，同一依赖的多条记录合并为一行。

### 3.3 通用样式规范

| 元素 | 样式 |
|------|------|
| 标题行（概览） | 深蓝底、白色加粗 16pt、水平居中 |
| 表头行（漏洞/依赖） | 皇家蓝底、白色加粗、水平居中、细边框 |
| 数据行 | 垂直居中、细边框、自动换行 |
| 列宽 | 自动调整（`sheet.autoSizeColumn`） |

### 3.4 适用场景

- 团队漏洞跟踪和管理
- 导入 JIRA / TAPD 等项目管理工具
- 制作漏洞修复排期表
- 管理层汇报数据汇总

### 3.5 API 访问

```
GET /api/reports/{taskId}?format=excel
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
Content-Disposition: attachment; filename="dependency-check-report-{id}.xlsx"
```

## 4. PDF 报告

### 4.1 生成方式

PDF 报告通过 **Flying Saucer** 库（`ITextRenderer`）将已有的 HTML 报告转换为 PDF：

```
HTML 报告（引擎生成）
       │
       ▼  Files.readString()
HTML 内容字符串
       │
       ▼  ITextRenderer.setDocumentFromString()
Flying Saucer 渲染引擎
       │
       ▼  renderer.layout() + createPDF()
PDF 文件
```

### 4.2 特性

| 特性 | 说明 |
|------|------|
| 内容一致性 | 与 HTML 报告内容完全一致（同源转换） |
| 中文支持 | 通过 Flying Saucer 内置字体支持中文 |
| 离线可读 | 不依赖浏览器，适合归档 |
| 固定布局 | 分页清晰，适合打印 |

### 4.3 内容结构

与 HTML 报告完全一致（参见第 2 节），但去除了交互式导航，转为线性排版：

- 封面：报告标题 + 项目信息
- 摘要页：依赖/漏洞统计总览
- 漏洞清单：按严重等级降序排列
- 许可证列表

### 4.4 适用场景

- 合规审计归档
- 邮件发送给非技术干系人
- 打印留存
- 正式文档交付

### 4.5 API 访问

```
GET /api/reports/{taskId}?format=pdf
Content-Type: application/pdf
Content-Disposition: attachment; filename="dependency-check-report-{id}.pdf"
```

## 5. 报告存储策略

### 5.1 存储架构

```
                 扫描完成
                    │
          ┌─────────┼─────────┐
          ▼         ▼         ▼
       HTML      Excel      PDF
      报告生成   报告生成    报告生成
          │         │         │
          └────┬────┴────┬────┘
               ▼         ▼
          本地文件系统  阿里云 OSS
          ./reports/   （条件装配）
               │         │
               └────┬────┘
                    ▼
              由 ReportController
              统一对外提供服务
```

### 5.2 两层存储策略

| 层级 | 说明 |
|------|------|
| **本地存储（主）** | `./reports/{taskId}/` 目录，始终可用 |
| **OSS 存储（辅）** | 仅当 `app.oss.enabled=true` 时启用，上传失败不影响报告生成 |

### 5.3 报告获取流程

```
请求报告
    │
    ├─ 1. 本地文件存在 → 直接返回（快速路径）
    │
    ├─ 2. 本地不存在 + OSS 可用 → 从 OSS 下载到本地（回源）
    │
    └─ 3. 都不存在 → 返回"报告文件不存在"
```

> 💡 对前端调用方完全透明——无论本地存储还是 OSS，API 调用方式完全相同。

### 5.4 文件命名规范

| 格式 | 文件名 |
|------|--------|
| HTML | `dependency-check-report.html` |
| Excel | `dependency-check-report-{taskId}.xlsx` |
| PDF | `dependency-check-report-{taskId}.pdf` |

## 6. 报告 API 接口

### 6.1 接口总览

| 方法 | 路径 | 说明 | 返回 |
|------|------|------|------|
| GET | `/api/reports/{taskId}?format=html` | 查看 HTML 报告 | `text/html`（inline） |
| GET | `/api/reports/{taskId}?format=excel` | 下载 Excel 报告 | `xlsx`（attachment） |
| GET | `/api/reports/{taskId}?format=pdf` | 下载 PDF 报告 | `pdf`（attachment） |
| GET | `/api/reports/{taskId}/exists` | 检查报告是否存在 | `Result<Boolean>` |
| GET | `/api/reports/{taskId}/formats` | 查询可用格式列表 | `Result<{taskId, formats[]}>` |

### 6.2 统一响应格式

所有 API 响应遵循 `Result<T>` 包装格式：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

文件下载类接口（HTML/Excel/PDF）直接返回文件流，不使用 JSON 包装。

### 6.3 格式参数

| 参数值 | 格式 | Content-Disposition |
|--------|------|---------------------|
| `html`（默认） | `text/html` | `inline`（浏览器预览） |
| `excel` | `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` | `attachment`（强制下载） |
| `pdf` | `application/pdf` | `attachment`（强制下载） |

## 7. 格式对比与选择建议

| 维度 | HTML | Excel | PDF |
|------|------|-------|-----|
| **信息完整度** | ⭐⭐⭐ 最全 | ⭐⭐ 结构化 | ⭐⭐⭐ 与 HTML 一致 |
| **可读性** | ⭐⭐⭐ 交互式 | ⭐⭐ 表格式 | ⭐⭐ 线性 |
| **可编辑性** | ⭐ 不易编辑 | ⭐⭐⭐ 自由编辑 | ⭐ 不可编辑 |
| **可分享性** | ⭐⭐ 需浏览器 | ⭐⭐⭐ 通用 | ⭐⭐⭐ 通用 |
| **归档适性** | ⭐ 不适合 | ⭐⭐ 适合 | ⭐⭐⭐ 最适 |
| **生成耗时** | 快（引擎原生） | 中（POI 构建） | 慢（HTML→PDF转换） |
| **文件大小** | 小（~100KB） | 中（~50KB） | 小（~200KB） |

### 选择建议

```
日常开发查看 → HTML
  └── 浏览器打开，交互式导航，信息最全

团队协作跟踪 → Excel
  └── 可编辑、可过滤、可导入 JIRA

正式交付归档 → PDF
  └── 不可篡改、不分平台、适合打印
```
