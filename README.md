## 一、产品概述

### 1.1 项目背景

集团内部已累计数万款Java、前端及多语言应用，第三方组件依赖数量呈指数级增长。近一年因第三方依赖漏洞导致的安全事件占比超60%，同时许可证不合规问题可能引发知识产权纠纷和重大商业损失。

目前集团内部缺乏统一的依赖分析工具和标准化使用文档，各团队使用的零散工具存在功能不全、配置复杂、缺乏维护等问题。本项目以业界成熟的开源安全引擎为基础，通过深度复现、文档完善和轻量适配，打造一套可直接部署使用的内部解决方案。

### 1.2 产品定位

面向集团内部所有开发团队的轻量级、标准化依赖安全与合规分析工具，提供开箱即用的依赖漏洞检测、许可证合规分析能力，同时配套完整的使用和开发文档，大幅降低工具使用门槛。

### 1.3 核心价值

1. **业务价值**：统一集团依赖分析标准，提前发现并修复依赖安全漏洞和许可证合规风险，保障应用稳定性和业务合规性
2. **文档价值**：输出完整文档集，解决集团内部缺乏标准化依赖分析工具文档的痛点
3. **技术价值**：基于业界成熟开源引擎构建，代码质量高、可扩展性强，为后续功能扩展奠定基础

## 三、产品功能需求

### 3.1 基准引擎核心能力

产品必须完整复现OWASP Dependency-Check 9.0.0的所有核心能力，并完成针对集团环境的适配优化：

1. **多语言依赖解析能力**
   - 支持Java（Maven/Gradle）、JavaScript（NPM）、Python等主流语言的依赖解析
   - 支持单模块和多模块项目的依赖树构建
   - 支持排除指定路径或依赖的扫描

2. **漏洞检测能力**
   - 集成NVD、OSV等官方漏洞数据源
   - 支持漏洞数据库的全量同步和增量更新
   - 支持漏洞等级划分（高危、中危、低危）和详情展示
   - 支持漏洞白名单配置，排除误报或无需修复的漏洞

3. **许可证合规分析能力**
   - 自动识别依赖组件的开源许可证类型
   - 检测许可证冲突和不合规使用情况
   - 生成许可证合规报告

4. **集团环境适配优化**
   - 针对集团网络环境优化漏洞数据库同步配置
   - 针对集团主流技术栈（尤其是阿里技术栈）优化扫描参数
   - 制定集团标准扫描配置模板，提升扫描速度和准确率

### 3.2 轻量Web服务能力

基于SpringBoot构建轻量级Web服务，提供可视化的依赖分析能力：

1. **项目管理能力**
   - 支持上传ZIP格式的项目压缩包
   - 支持自定义扫描参数（排除路径、报告格式等）

2. **扫描任务管理能力**
   - 支持扫描任务的提交、取消和状态查询
   - 支持扫描任务异步执行，避免阻塞主线程
   - 支持扫描进度的实时展示

3. **报告管理能力**
   - 支持生成HTML、PDF、Excel三种格式的扫描报告
   - 报告内容包含集团定制化信息
   - 支持报告的在线预览和下载
   - 报告存储在阿里云OSS，支持长期保存

4. **数据管理能力**
   - 支持扫描历史记录查询
   - 支持扫描结果统计（漏洞数量、许可证问题数量等）
   - 支持扫描结果导出
   - 支持漏洞详情查询

### 3.3 前端交互能力

开发极简前端页面，实现完整的用户交互流程：

1. **用户认证**：支持简单的用户登录（固定账号密码，无需注册功能）

2. **核心页面**
   - 项目上传与扫描任务提交页面
   - 扫描任务列表页面（显示任务状态和操作按钮）
   - 扫描结果详情页面（展示漏洞列表和许可证问题）
   - 报告下载页面

3. **交互要求**：页面简洁易用，交互流畅，无明显卡顿和Bug

### 3.4 部署与运维能力

1. 提供完整的Docker部署方案，支持一键部署
2. 支持前后端分离部署，使用Nginx作为反向代理
3. 系统部署后可通过公网访问，稳定运行

## 四、非功能需求

### 4.1 性能要求

1. 小型项目（代码量<1万行）扫描时间≤30秒
2. 中型项目（代码量1-10万行）扫描时间≤5分钟
3. 大型项目（代码量>10万行）扫描时间≤30分钟
4. 系统支持同时处理至少5个并发扫描任务
5. 扫描过程中内存占用≤4GB

### 4.2 兼容性要求

1. 支持JDK17及以上版本运行
2. 支持主流浏览器（Chrome、Firefox、Edge）访问
3. 兼容集团主流技术栈（Spring Boot、Spring Cloud Alibaba等）

### 4.3 易用性要求

1. 提供完整的用户使用手册，包含详细的操作步骤和截图
2. 系统界面简洁直观，用户无需培训即可上手使用
3. 提供常见问题排查手册，帮助用户快速解决使用过程中遇到的问题

### 4.4 安全性要求

1. 系统仅用于集团内部使用，禁止对外开放
2. 禁止上传包含敏感信息的项目代码
3. 扫描报告仅对提交任务的用户可见
4. 严格遵守OWASP Dependency-Check的Apache 2.0开源协议

### 4.5 可维护性要求

1. 核心方法和类必须添加详细注释
2. 提供完整的开发手册，便于后续维护和功能扩展
3. 配置参数集中管理，支持动态更新，无需重启服务

## 七、风险与注意事项

1. **网络风险**：NVD漏洞数据库同步可能因网络问题失败，需提前准备离线同步方案或使用国内镜像
2. **性能风险**：扫描大型项目可能出现内存溢出，需提前调整JVM参数，优化扫描配置
3. **适配风险**：部分阿里技术栈组件可能无法被准确识别，需记录问题并给出临时解决方案
4. **开源协议风险**：严格遵守OWASP Dependency-Check的Apache 2.0开源协议，不得用于商业用途
