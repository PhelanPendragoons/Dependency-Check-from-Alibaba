#### B4-05#4 — ZIP 与项目目录分离

> **核查发现**：当前 [L158](dependency-check-web/src/main/java/com/alibaba/dependencycheck/service/ProjectService.java#L158) `zipPath = projectDir + File.separator + originalFilename` 将 ZIP 存在项目目录内部。若 ZIP 被占用删不掉 → 父目录无法递归删除 → 整个 `./uploads/{name}/` 残留。改用 `originalFilename` 有特殊字符风险；改用 `{name}.zip` 利用已有的 `validateProjectName()` 安全保障。

- [ ] **B4-05#4-01** 新增常量 `private static final String ZIP_DIR = "./uploads/_zips";`
- [ ] **B4-05#4-02** `createProject()` [L158](dependency-check-web/src/main/java/com/alibaba/dependencycheck/service/ProjectService.java#L158) 改造：
  ```java
  // 改前: String zipPath = projectDir + File.separator + originalFilename;
  // 改后: ZIP 独立存储，与解压目录物理隔离
  Path zipsDir = Paths.get(ZIP_DIR);
  if (!Files.exists(zipsDir)) { Files.createDirectories(zipsDir); }
  String zipPath = ZIP_DIR + File.separator + name + ".zip";
  ```

#### #105 — 任务取消功能

> **核查发现**：当前 `ScanTaskController` 无 CANCEL 接口。`ScanTaskExecutorService.executeScan()` 是 `@Async void` 方法 — 无返回值，无法直接获取 `Future`。取消必须通过：① `ConcurrentHashMap<Long, Future<?>>` 跟踪活跃扫描 → ② `cancelTask()` 查表调 `Future.cancel(true)` → ③ `executeScan()` 的 catch 块先读 DB 状态再决定覆盖为 FAILED 还是保留 CANCELLED。

- [ ] **#105-01** `ScanTaskController` 新增 `POST /api/tasks/{id}/cancel` 接口
  ```java
  @PostMapping("/{id}/cancel")
  public Result<ScanTaskDTO> cancelTask(@PathVariable Long id) {
      ScanTaskDTO task = scanTaskService.cancelTask(id);
      return Result.success(task);
  }
  ```
- [ ] **#105-02** `ScanTaskService` 新增 `cancelTask(Long taskId)` 方法：
  - 检查任务存在性（不存在 → BusinessException）
  - 仅 PENDING / RUNNING 状态可取消（COMPLETED / FAILED / CANCELLED → BusinessException）
  - 调用 `scanTaskExecutorService.cancelScan(taskId)` 中断执行线程
  - 更新 DB 状态为 CANCELLED
- [ ] **#105-03** `ScanTaskExecutorService` 新增取消机制：
  - 新增 `ConcurrentHashMap<Long, Future<?>> activeScans` 维护活跃扫描
  - `executeScan()` 开始时 `activeScans.put(taskId, Future)` → 结束时 `activeScans.remove(taskId)`
  - 新增 `cancelScan(Long taskId)` 方法：查 `activeScans` → `future.cancel(true)` 中断线程
  - **关键**：catch 块 [L118](dependency-check-web/src/main/java/com/alibaba/dependencycheck/service/ScanTaskExecutorService.java#L118) 中**先读 DB 确认当前状态**，若已是 CANCELLED 则不覆盖为 FAILED
- [ ] **#105-04** 编写任务取消功能的单元测试（`ScanTaskServiceTest` 新增约 4 个用例）：
  - PENDING 任务取消成功
  - RUNNING 任务取消成功（Mock Future）
  - COMPLETED 任务取消失败（预期抛异常）
  - 不存在的任务取消失败
- [ ] **验证**：`mvn test` 全量回归，确保 46 + 新增 ≥ 4 = ≥ 50 测试通过
