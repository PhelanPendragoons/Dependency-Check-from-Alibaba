## 1. 系统概述

### 1.1 系统简介

应用依赖安全与合规分析平台是一套轻量级依赖安全扫描工具，基于 OWASP Dependency-Check 引擎构建。您可以通过 Web 界面上传项目源码（ZIP 格式），系统会自动分析项目的第三方依赖，检测已知安全漏洞（CVE）和开源许可证合规问题，并生成多格式分析报告。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 项目上传 | 支持上传 ZIP 格式的项目源码包 |
| 漏洞扫描 | 自动解析依赖，匹配 NVD/OSV 漏洞数据库 |
| 许可证分析 | 识别依赖组件的开源许可证类型 |
| 报告生成 | 支持 HTML、Excel、PDF 三种格式 |
| 仪表盘 | 聚合展示项目数、漏洞统计、等级分布 |

### 1.3 扫描支持的依赖类型

- **Java**：Maven（pom.xml）、Gradle（build.gradle）
- **JavaScript**：NPM（package.json）
- **Python**：pip（requirements.txt）、Pipfile
- 以及更多语言（由 OWASP Dependency-Check 引擎提供）

## 3. 快速开始

### 3.1 访问系统

在浏览器地址栏输入系统地址，进入首页仪表盘：

```
http://[服务器地址]:[端口]
```

> 📷 **截图占位**：系统首页仪表盘

### 3.2 界面布局

系统界面分为三个区域：

| 区域 | 位置 | 说明 |
|------|------|------|
| **侧边栏** | 左侧深色区域 | 导航菜单：仪表盘、项目管理、扫描任务、报告中心 |
| **顶栏** | 顶部白色区域 | 面包屑导航、全屏按钮、用户信息 |
| **主内容区** | 中间大面积区域 | 当前页面的核心操作区域 |

### 3.3 核心操作流程

一次完整的依赖安全扫描只需 4 步：

```
上传项目 → 开始扫描 → 查看结果 → 下载报告
```

下面逐页说明每个页面的使用方法。

## 4. 仪表盘

仪表盘是登录后的首页，集中展示平台的统计概况。

### 4.1 统计卡片

页面顶部有 **4 个统计卡片**：

| 卡片 | 显示内容 |
|------|----------|
| **项目总数** | 已上传的项目数量 |
| **扫描任务** | 历史扫描任务总数 |
| **漏洞总数** | 所有扫描发现的安全漏洞总数 |
| **高危 / 严重** | CRITICAL + HIGH 等级的漏洞数量之和 |

### 4.2 漏洞等级分布

当系统中有漏洞数据时，仪表盘会显示 **4 个等级卡片**：

| 等级 | 颜色标识 | 说明 |
|------|----------|------|
| CRITICAL（严重） | 暗红色 | CVSS ≥ 9.0，需立即修复 |
| HIGH（高危） | 红色 | CVSS 7.0-8.9，需尽快修复 |
| MEDIUM（中危） | 橙色 | CVSS 4.0-6.9，建议修复 |
| LOW（低危） | 绿色 | CVSS < 4.0，可延后处理 |

### 4.3 最近扫描任务

仪表盘下方展示最近 10 条扫描任务，包含任务 ID、状态、依赖数量、漏洞数量和进度条。

> 📷 **截图占位**：仪表盘全貌，标注四个统计卡片 + 等级分布 + 最近任务

## 5. 项目管理

点击左侧菜单 **「项目管理」** 进入项目列表。

### 5.1 项目列表

列表展示所有已上传的项目，包含以下信息：

| 列 | 说明 |
|----|------|
| ID | 项目编号（系统自动分配） |
| 项目名称 | 上传时填写的名称，点击可进入详情 |
| 描述 | 项目描述信息 |
| 文件类型 | 上传的文件格式（ZIP） |
| 状态 | UPLOADED = 已上传，可发起扫描 |
| 创建时间 | 项目上传时间 |
| 操作 | 「开始扫描」和「删除」按钮 |

列表底部支持分页，可切换每页显示 5/10/20 条。

> 📷 **截图占位**：项目列表页

### 5.2 上传项目

1. 点击页面右上角 **「上传项目」** 按钮
2. 在弹出的对话框中填写信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 项目名称 | ✅ 必填 | 自定义项目名称，建议与源码仓库同名 |
| 项目描述 | 选填 | 补充说明项目用途或扫描目的 |
| 上传文件 | ✅ 必填 | 选择 `.zip` 格式的项目源码压缩包 |

3. 点击 **「上传并创建」**
4. 看到绿色提示「项目创建成功」即完成

> 📷 **截图占位**：上传项目对话框

### 5.3 查看项目详情

在项目列表中点击项目名称，进入项目详情页：

- 顶部展示项目基本信息（ID、名称、描述、文件路径）
- 下方展示该项目的**扫描历史**列表
- 可在此页面点击 **「开始扫描」** 直接发起新扫描

> 📷 **截图占位**：项目详情页

### 5.4 删除项目

在项目列表的操作列点击 **「删除」** → 确认弹窗点「确定」→ 项目及其关联文件被删除。

> ⚠️ 删除项目会同时删除该项目下的所有扫描结果和报告，请谨慎操作。

## 6. 扫描任务管理

点击左侧菜单 **「扫描任务」** 进入任务列表。

### 6.1 创建扫描任务

有两种方式创建扫描任务：

**方式一（推荐）**：在「项目管理」页面，找到目标项目，点击操作列的 **「开始扫描」**。

**方式二**：在「项目详情」页面，点击 **「开始扫描」** 按钮。

创建成功后，页面提示「扫描任务已创建」，系统自动跳转到任务列表。

### 6.2 扫描状态说明

一个扫描任务会经历以下状态流转：

```
PENDING → RUNNING → COMPLETED （成功）
                   → FAILED    （失败）
PENDING / RUNNING  → CANCELLED （手动取消）
```

| 状态 | 标签颜色 | 说明 |
|------|----------|------|
| PENDING | 灰色 | 任务已创建，等待执行 |
| RUNNING | 橙色 ⚠ | 正在扫描中，页面自动刷新 |
| COMPLETED | 绿色 ✅ | 扫描完成，可查看结果 |
| FAILED | 红色 ❌ | 扫描失败，可查看错误信息 |
| CANCELLED | 灰色 | 用户手动取消 |

### 6.3 自动刷新

当列表中存在 RUNNING 或 PENDING 状态的任务时，系统会**自动每隔 3 秒刷新**任务状态，页面右上角显示「🔄 自动刷新中」标签。您无需手动刷新浏览器。

扫描完成后（所有任务不再是 RUNNING/PENDING），自动刷新会自动停止。

> 📷 **截图占位**：任务列表，标注自动刷新标签和进度条变化

### 6.4 任务列表

| 列 | 说明 |
|----|------|
| 任务 ID | 系统自动编号 |
| 项目 ID | 对应项目的编号 |
| 状态 | PENDING / RUNNING / COMPLETED / FAILED / CANCELLED |
| 依赖总数 | 扫描发现的依赖包总数 |
| 漏洞数 | 存在已知 CVE 漏洞的依赖数 |
| 进度 | 进度条（10% → 90% → 100%） |
| 开始时间 | 扫描开始时间 |
| 完成时间 | 扫描结束时间 |
| 操作 | 「详情」和「取消」按钮 |

列表底部支持分页。

### 6.5 取消扫描

对于 RUNNING 或 PENDING 状态的任务，可点击操作列的 **「取消」** 按钮中止扫描。取消后任务状态变为 CANCELLED。

> ⚠️ COMPLETED 和 FAILED 状态的任务无法取消。

## 7. 查看扫描结果

### 7.1 进入任务详情

在「扫描任务」列表点击 **「详情」** 按钮，进入任务详情页。

### 7.2 任务详情页

页面展示任务的完整信息：

| 信息项 | 说明 |
|--------|------|
| 项目 ID | 对应项目的编号 |
| 状态 | 任务当前状态标签 |
| 总依赖数 | 扫描发现的所有依赖包数量 |
| 漏洞依赖数 | 存在已知漏洞的依赖包数量 |
| 进度 | 最终进度（COMPLETED = 100%） |
| 开始时间 | 任务开始执行的时间 |
| 完成时间 | 任务执行完毕的时间 |
| 错误信息 | 仅 FAILED 状态显示（红色警告） |

> 📷 **截图占位**：任务详情页（COMPLETED 状态）

### 7.3 扫描结果解读

任务完成后，系统自动分析所有依赖的漏洞和许可证信息，并在报告中展示：

- **CVE 编号**：Common Vulnerabilities and Exposures 标准编号
- **CVSS 评分**：0.0-10.0，分数越高越危险
- **严重等级**：CRITICAL / HIGH / MEDIUM / LOW
- **影响版本**：存在漏洞的依赖版本范围
- **修复建议**：建议升级的安全版本

## 8. 报告下载

### 8.1 进入报告中心

点击左侧菜单 **「报告中心」**，查看所有已完成扫描的项目。

### 8.2 支持的报告格式

对于每个 COMPLETED 状态的扫描任务，系统提供三种格式的报告下载：

| 格式 | 按钮 | 适用场景 |
|------|------|----------|
| **HTML** | 蓝色「HTML」按钮 | 在线查看，浏览器直接打开，包含交互式导航 |
| **Excel** | 绿色「Excel」按钮 | 团队分享、漏洞跟踪表、导入 JIRA |
| **PDF** | 橙色「PDF」按钮 | 归档留存、合规审计、邮件发送 |

> 📷 **截图占位**：报告中心页面，标注三种格式按钮

### 8.3 查看 HTML 报告

点击 **「HTML」** → 浏览器新标签页打开完整的漏洞分析报告，包含：
- 扫描摘要（项目名、时间、依赖/漏洞统计）
- 漏洞清单（CVE 编号、CVSS 评分、严重等级、影响组件）
- 许可证列表

### 8.4 下载 Excel 报告

点击 **「Excel」** → 自动下载 `.xlsx` 文件，包含三个工作表：
- **漏洞清单**：详细的 CVE 列表
- **依赖汇总**：所有依赖包列表
- **许可证信息**：许可证合规状态

### 8.5 下载 PDF 报告

点击 **「PDF」** → 自动下载 `.pdf` 文件，格式适合打印和归档。

### 8.6 从任务详情页下载

也可以在「任务详情」页（COMPLETED 状态），直接点击底部的报告下载按钮。

> 📷 **截图占位**：HTML 报告预览页，标注漏洞列表区域

## 9. 常见问题

### Q1：上传后点击扫描，扫描很慢怎么办？

A：扫描速度取决于项目大小和 NVD 数据库状态：
- 小型项目（<1万行代码）：约 30 秒
- 中型项目（1-10万行代码）：1-5 分钟
- 首次扫描时需要下载 NVD 数据（取决于网络），后续扫描使用缓存

### Q2：为什么扫描结果中漏洞数量为 0？

A：可能原因：
1. 项目的依赖版本较新，暂无已知漏洞 — 这是好事
2. 项目依赖很少，未被 NVD 数据库收录
3. 项目使用的依赖管理工具不在支持列表中

### Q3：漏洞等级是如何划分的？

A：依据 CVSS v3.x 评分标准：
- CRITICAL：9.0 - 10.0
- HIGH：7.0 - 8.9
- MEDIUM：4.0 - 6.9
- LOW：0.1 - 3.9

### Q4：报告下载后打不开？

A：
- HTML 报告：用浏览器直接打开
- Excel 报告：需要 Microsoft Excel 或 WPS 打开
- PDF 报告：需要 PDF 阅读器（Adobe Reader / 浏览器内置）

### Q5：可以同时扫描多个项目吗？

A：可以。系统支持最多 4 个并发扫描任务（由线程池配置控制）。同一项目的重复扫描请求会被拦截，需等当前任务完成后再提交。

### Q6：任务失败怎么办？

A：在任务详情页查看**红色错误信息**，常见失败原因：
- 项目 ZIP 文件损坏或格式不正确
- 项目目录中无支持的依赖文件
- NVD 数据库同步异常
- 磁盘空间不足

解决后可在项目管理页重新发起扫描。

### Q7：扫描报告保存在哪里？

A：报告默认存储在服务器本地文件系统 `./reports/` 目录。若系统配置了阿里云 OSS（生产环境），报告会自动上传至 OSS 实现持久化存储。
