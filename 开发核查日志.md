## 第三轮：安全与性能审查

### 3.1 安全审查
- [ ] **安全 1**：**CORS 配置过于宽松** — `addAllowedOriginPattern("*")` 允许所有来源，且 `setAllowCredentials(true)` 与通配符 Origin 冲突
- [ ] **安全 2**：**文件上传路径遍历** — Zip Slip 防护已实现 ✅，但未检查文件名中的 Windows 保留字符
- [ ] **安全 3**：**敏感信息泄露** — `errorMessage` 可能包含文件路径、API Key 等敏感信息
- [ ] **安全 4**：**XSS 防护** — 报告中的依赖名称/CVE 描述直接来自 DependencyCheck 输出，如果包含恶意 HTML 脚本，浏览器会执行
- [ ] **安全 5**：**CSRF 防护** — Spring Security 未引入，所有 API 都没有 CSRF Token 保护
- [ ] **安全 6**：**速率限制** — API 没有限流保护，恶意用户可大量创建扫描任务耗尽服务器资源

### 3.2 性能审查
- [ ] **性能 1**：**大文件上传** — 500MB 文件上传时，`transferTo()` 会写入临时目录，磁盘 I/O 可能成为瓶颈
- [ ] **性能 2**：**并发扫描** — 线程池最大 4 个线程，同时扫描 4 个项目时，每个扫描任务都会创建独立的 `Engine` 实例，内存消耗大
- [ ] **性能 3**：**全表查询** — `listProjects()` 使用 `selectList(null)`，项目数量多时性能差
- [ ] **性能 4**：**NVD 更新** — 每次扫描都创建新的 `Engine` 实例，如果 NVD 数据库需要更新，会阻塞扫描线程

### 3.3 日志审查
- [ ] **日志 1**：`log.info("上传项目: name={}, fileName={}, size={}")` — 记录文件大小，合理 ✅
- [ ] **日志 2**：`log.error("扫描失败", e)` — 记录完整堆栈，合理 ✅
- [ ] **日志 3**：`log.warn("检测到路径穿越攻击，已跳过: {}")` — 记录安全事件，合理 ✅
- [ ] **日志 4**：MyBatis-Plus 的 `StdOutImpl` 会打印所有 SQL，生产环境建议改为 `Slf4jImpl`
